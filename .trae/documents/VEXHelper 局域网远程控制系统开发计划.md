# VEXHelper 局域网远程投屏与控制系统开发计划 (修订版)

根据您的最新指示，我们将系统重新定义为“手机端控制，浏览器端显示与扩音”的模式。

## 1. 系统架构 (Client-Server Model)
- **角色定义**：
  - **iOS App (Server/Controller)**：作为主机，负责计时逻辑、发送指令、托管 Web 服务。
  - **浏览器 (Client/Display)**：作为纯显示端（大屏），负责显示时间和**播放声音**。
- **通信协议**：
  - **WebSocket**：iOS 向浏览器发送 `updateTime`（时间同步）、`playSound`（播放声音）指令。
  - **HTTP**：iOS 提供 HTML 页面和 **MP3 音频文件** 下载。

## 2. 浏览器端开发 (Web Display)
打造一个纯净的“大屏显示器”。
- **界面设计**：
  - **无交互**：移除所有控制按钮，仅保留全屏时间显示和状态指示。
  - **视觉风格**：复刻 iOS 端的液态/磨砂玻璃风格，支持深色模式。
- **音频系统**：
  - **音频托管**：通过 `<audio>` 标签预加载 iOS 端提供的音频文件 (`Start.MP3`, `Stop.MP3`, `Over.MP3`, `Change.MP3`)。
  - **指令播放**：监听 WebSocket 的 `playSound` 事件触发播放。
  - **用户交互**：添加“点击以启用声音”的遮罩层（解决浏览器自动播放策略限制）。

## 3. iOS 端开发 (Host & Control)
- **网络服务 (`LocalNetworkService`)**：
  - **静态资源服务**：除了 serving HTML/JS，还需暴露 `/audio/Start.MP3` 等路由供浏览器加载。
- **声音控制中心改造**：
  - 修改 `SoundsControlCenter`，增加“远程广播模式”。
  - **逻辑变更**：当远程模式开启时，**拦截本地音频播放**，改为发送 WebSocket 指令 `{ "type": "playSound", "file": "Start" }`。
- **UI 调整**：
  - **控制界面**：复用现有的计时器界面作为核心控制器。
  - **连接管理页**：在设置中添加“远程显示配置”页面，包含：
    - 服务开关。
    - 投屏地址/二维码。
    - “仅浏览器播放声音”的选项开关（默认开启）。

## 4. 实施步骤
1.  **网络层构建**：搭建支持文件传输（MP3/HTML）的 `NWListener` 服务器。
2.  **Web端实现**：编写包含音频播放逻辑的 `index.html` 和 `app.js`。
3.  **音频路由改造**：在 `TimerEngine` 和 `SoundsControlCenter` 中截获播放事件并转发。
4.  **设置页集成**：添加服务器启动入口和二维码显示。
